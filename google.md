# Gemini Live Agent Challenge Checklist

## 1. Application Requirements
- [x] **NEW Project**: Created newly during the Contest Period (not a modification of prior/existing work).
- [x] **Next-Generation AI Agent**: Moves beyond simple text-in/text-out interactions utilizing multimodal inputs and outputs.
- [x] **Gemini Model**: Project explicitly leverages a Gemini model.
- [x] **Development Kit**: Built using the Google GenAI SDK or Agent Development Kit (ADK).
- [x] **Google Cloud Services**: Uses at least one Google Cloud service.
- [x] **Acceptable Use Policy**: Adheres to the Google Cloud AUP.
- [x] **Category Selected** (Must map specifically to one of these):
  - [x] **Live Agents**: Real-time interaction (Audio/Vision) that can be interrupted (Must use Gemini Live API / ADK).
  - [ ] **Creative Storyteller**: Multimodal storytelling seamlessly weaving text, images, audio, video in a fluid interleaved output stream.
  - [ ] **UI Navigator**: Visual UI understanding (becomes the user's hands on screen) interpreting screenshots/displays into executable actions.
- [ ] **Team Setup**: All Eligible Individuals added as members of the Project on Devpost (with a designated Representative if a team).
- [x] **Functionality**: Capable of being successfully installed and consistently run exactly as depicted in the video/description.
- [x] **Third-Party Integrations**: Fully legally authorized to use any 3rd-party APIs/Data. (Must be listed with specificity in the submission description).
- [ ] **Testing Access**: App provided free of charge without restriction for judging (via link, demo, or test build; include credentials if private).
- [x] **English Language Requirement**: App UI, Submission materials, video, text description, and instructions support English.
- [x] **Intellectual Property**: Original work, solely owned, with no trademark/copyright violations. Grants Sponsor necessary promotional usage rights.

## 2. Submission Details & Assets
- [ ] **Category Tagging**: Labeled correctly on the submission form.
- [ ] **Text Description**: Includes summary of features, tech used, data sources, findings, and learnings.
- [ ] **Public Code Repository**: URL provided and set to public access.
- [ ] **Spin-up Instructions**: Step-by-step specific guide in your `README.md` to set up locally or deploy to cloud.
- [ ] **Proof of Google Cloud Deployment** (Choose one): 
  - [ ] Quick screen recording showing backend running on GCP (console logs / deployment view).
  - [ ] Link directly to a code file in your repository demonstrating clear GCP Service/API calls (e.g., Vertex AI endpoints).
- [ ] **Architecture Diagram**: Clear visual representation of how your system connects (Gemini to Backend, DB, Frontend, etc.).
- [ ] **Demonstration Video** (Max length: 4 minutes):
  - [ ] Uploaded and publicly visible on YouTube or Vimeo.
  - [ ] Shows the Multimodal/Agentic project working in real-time (No mockups; show actual working software).
  - [ ] Pitch included explaining the problem being solved and the solution's value.
  - [ ] Audio/Subtitles strictly in English.
  - [ ] Contains no offensive, defamatory, unlawful content, or 3rd-party unapproved advertising/logos.

## 3. Optional Developer Contributions (Bonus Points)
- [ ] **Publishing Content (Blog/Podcast/Video)**: Documented how the project was built using Google AI/Cloud on a public platform (Medium, Dev.to, YouTube).
  - [ ] Content is fully Public (not unlisted).
  - [ ] Contains specific language/disclaimer stating it was created for the purposes of entering this hackathon.
  - [ ] Shared on social media using the exact hashtag: `#GeminiLiveAgentChallenge`.
- [ ] **Automating Cloud Deployment**: Shown via scripts or Infrastructure-as-Code (IaC) tools included directly within the public repository.
- [ ] **Google Developer Group (GDG) Membership**: Provided public profile link to demonstrate active GDG participation.
